DHCP & HA: Arquitectura de Red Crítica

Guía Autorizada para Administradores de Sistemas y la Continuidad del Servicio

1. Fundamentos del Dynamic Host Configuration Protocol (DHCP)

DHCP (RFC 2131) es el protocolo de red que automatiza la configuración de direcciones IP y otros parámetros esenciales (Máscara, Gateway, DNS, NTP) en dispositivos cliente. Su principal valor es la escalabilidad y la prevención de conflictos de IP.

El ciclo de cuatro pasos para obtener una asignación de IP:

  • Discover: Cliente busca servidor (Broadcast).
  • Offer: Servidor ofrece una configuración de IP.
  • Request: Cliente solicita la IP ofrecida.
  • Acknowledgment (ACK): Servidor confirma la asignación.

2. Estrategias de Direccionamiento IP: Estático vs. Reservas DHCP

La IP estática se reserva para dispositivos que requieren máxima fiabilidad y no deben depender del estado de un servidor DHCP. Es fundamental para:

  • **Nodos de Infraestructura:** Routers, Firewalls, Switches L3 y Servidores Host.
  • **Servicios Definidos:** Equipos vinculados a políticas de seguridad (ej. RADIUS/NPS).
Requiere IPAM estricto para mitigar el riesgo de IPs duplicadas.

Ofrece la previsibilidad de una IP constante con la capacidad de gestión centralizada de DHCP. Ideal para:

  • Dispositivos de difícil acceso (WAPs, cámaras, impresoras) donde la administración remota es clave.
  • Permite el cambio masivo de opciones (DNS, NTP) sin interacción en el dispositivo final.

3. Optimización del Rendimiento del Servidor DHCP

Para reducir la huella de memoria, los administradores deben estructurar las opciones de configuración para que se hereden desde contenedores padre (Global > Red > Subred). Evitar la replicación explícita de opciones reduce la carga de datos del servidor.

El registro de eventos es una operación de E/S costosa. Para periodos de alta actividad:

  • **Práctica:** Limitar o deshabilitar los niveles verbosos (INFO, TRACE).
  • **Solución de Problemas:** Habilitar el registro detallado de forma **dinámica** solo durante la fase de *debugging*.

Estos parámetros controlan el *ping* ICMP preventivo que el servidor realiza antes de ofrecer una dirección. Su correcta configuración es vital para evitar la congestión en el procesamiento de mensajes DISCOVER.

Requisito de Rendimiento:

El valor de numprocessors debe exceder el número de mensajes DISCOVER que se puedan recibir en un intervalo de pingTime durante el pico de actividad.

4. Seguridad de DHCP y Mitigación de Amenazas

La capa de acceso de la red es vulnerable a ataques basados en DHCP:

  • **Servidor DHCP No Autorizado (Rogue DHCP Server):** Asigna configuraciones maliciosas, dirigiendo el tráfico a un atacante (Man-in-the-Middle).
  • **DHCP Starvation:** Agotamiento del pool de direcciones IP, causando Denegación de Servicio (DoS) para nuevos clientes legítimos.

DHCP Snooping es la técnica de seguridad a nivel de switch (Capa 2) para proteger la infraestructura, monitoreando y filtrando el tráfico DHCP.

Mecanismos de Protección:

  • **Control de Servidores:** Solo los puertos de servidores autorizados pueden enviar ofertas (OFFER, ACK).
  • **Filtro de Asignación:** Mantiene una tabla de *bindings* (IP/MAC) para asegurar que los hosts usen solo las direcciones asignadas legítimamente.

5. DHCP y la Alta Disponibilidad (HA) en Clústeres

La Alta Disponibilidad (HA) asegura que los servicios toleren fallos y mantengan la continuidad operativa. Para DHCP, esto significa garantizar que el servicio de asignación de IP esté disponible sin interrupción, incluso durante el fallo de un nodo.

Estructuras utilizadas para asegurar la disponibilidad y el rendimiento:

  • **HA (Redundancia):** Activo-Pasivo (foco en la disponibilidad).
  • **Carga Balanceada:** Activo-Activo (foco en el rendimiento y la escalabilidad).
  • **Híbrido:** La combinación más robusta, ideal para servicios críticos de una empresa.

El diseño HA debe ser integral:

  • **Entorno:** Redundancia de UPS, temperatura controlada, enlaces WAN redundantes.
  • **Almacenamiento (CRÍTICO):** La información compartida (*leases* de DHCP, estado del clúster) debe residir en **Almacenamiento Externo RAID** (SAN o NAS) accesible por todos los nodos.
  • **Arquitectura:** Nodos de dirección (HA) Activo-Pasivo y nodos de procesamiento (Carga) Activo-Activo.

Plataformas comerciales y de código abierto:

  • **Windows:** MSCS, NLB.
  • **Solaris:** Sun Cluster 3.1.
  • **Linux:** HA OSCAR, Ultra Monkey.
